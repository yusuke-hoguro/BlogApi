# ゴルーチンとチャネルの安全性について

## ルール
- チャネルを閉じることができるのは送信側（producer）のみ
- 受信側は閉じられたチャネルからデータを読み込む際に`ok`を確認すること
- 処理の中止が重要な場合は、チャネルの待機処理を常に`ctx.Done()` と組み合わせる
- バッファ付きチャネルは一時的なデータラッシュを吸収できるが、処理が遅い受信者の問題は解決しないので注意する

## 意味
- close(channel): これ以降、値は送信されなくなる
- ctx.Done(): 処理を直ちに停止する

## 検証結果
- 受信側が閉じられたチャネルを判定せずに受信し続けると、ゼロ値を高速に受信し続けて無限ループになる
- `ctx.Done()`が閉じると、バッファチャネルに未処理データが残っていても、`select`が`ctx.Done()`側を選べば`worker`は終了する。
